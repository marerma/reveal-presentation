HTML - basis for accessebility

Use the correct semantic HTML elements for their intended purpose as much as possible.

  • Clear and logical structure with <header>, <nav>, <main>, <ul>, <footer>

    • Why? Because screen readers can announce these tags with the same phrase and support hotkey navigation

Appropriate UI controls and Keyboard access

  • Use links, buttons, form elements, and labels.

  • Place the input element inside of a label element or use the label’s for attribute
  • Buttons and Links: meaningful content inside link text.
    Don't just use "Click here"
  • Mouse-specific events like mouseover, mouseout, dblclick should be doubled up with similar events
    • 									
										imgThumb.onmouseover = showImg;
										imgThumb.onmouseout = hideImg;

										imgThumb.onfocus = showImg;
										imgThumb.onblur = hideImg;

CSS - accessible styling

Font and Sizing

  • rem better than px to allow proper zooming
  • minimal font-size 18px
  • minimum reccomended size for touchable elements 48x48px
  • ensure that the elements are properly spaced out
  • the page is readable and functional when it is zoomed at least to 200%.

Color

  • Color is not used as the sole method of conveying content or distinguishing visual elements
  • Text and image contrast of at least 4.5:1

Invisible Content for screen-readers users

  • a skip navigation link (e.g. hidden until to the user navigates to it with a keyboard)
  • hidden elements, e.g. label for icons or additional text for breadcrumbs (prefer position: absolute than display:none and visibility: hidden)
    • 									
										.hidden-element {
											position:absolute;
											left:-10000px;
											top:auto;
											width:1px;
											height:1px;
											overflow:hidden;
										}

Media accessebility

  • alt-attribute for img: descriptive and not repeatative (good: white dog, bad: image of dog) or empty string for decorative icons
  • custom audio and video controls instead of default controls, which are not keyboard-accessible
  • provide text-transcripts for audio and video content
  • void video and animation that repeat or flash.
    • 								
									@media (prefers-reduced-motion: reduce) {
										button {
										  animation: none;
										}
									}

    WAI-ARIA

    • (Web Accessibility Initiative - Accessible Rich Internet Applications) is a specification written by the W3C
    • a set of additional HTML attributes
    • provide additional semantics and improve accessibility for custom elements
    • ARIA is suitable for dynamic content updates

      • Always favor semantic HTML elements where possible
